# Home 페이지 와이어프레임

## 현재 구현된 내용 분석

### 기존 구조
- **Header**: 네비게이션 메뉴, 검색바, 글쓰기 버튼, 사용자 아바타, PWA 설치 버튼
- **HeroSection**: 환영 메시지와 액션 버튼들이 포함된 카드 형태
- **MainContent**: 인기 게시글과 최근 게시글 목록 (PostsList 컴포넌트)
- **Footer**: 하단 푸터

### 데이터 구조
- **Post**: id, title, content, author, createdAt, views, comments, category, tags
- **Category**: id, name, icon, count
- **Tag**: name, count

## Sidebar 구성 요소

### 1. 카테고리 섹션
- **목적**: 게시글을 카테고리별로 분류하여 쉽게 찾을 수 있도록 함
- **구성**: 카테고리명, 아이콘, 게시글 수
- **예시**: 💻 개발 (15), 🎨 디자인 (8), 📱 모바일 (12) 등

### 2. 인기 태그 섹션
- **목적**: 자주 사용되는 태그들을 표시하여 관련 게시글을 쉽게 찾을 수 있도록 함
- **구성**: 태그명과 사용 횟수
- **예시**: #React (25), #TypeScript (18), #Next.js (15) 등

## 레이아웃 비율

- **MainContent**: 70-75% (게시글 목록)
- **Sidebar**: 25-30% (카테고리, 태그)

## 반응형 고려사항

### 데스크톱 (1024px 이상)
- Sidebar가 오른쪽에 고정 표시
- MainContent와 Sidebar가 가로로 배치

### 태블릿 (768px - 1023px)
- Sidebar가 MainContent 아래로 이동
- 세로로 배치되되, Sidebar 내용을 가로로 정렬

### 모바일 (768px 미만)
- Sidebar 내용을 접을 수 있는 아코디언 형태로 변경
- MainContent가 전체 너비 사용

## 구현 우선순위

1. **1단계**: 기본 Sidebar 컴포넌트 생성
2. **2단계**: 카테고리 섹션 구현
3. **3단계**: 인기 태그 섹션 구현
4. **4단계**: 반응형 레이아웃 적용
5. **5단계**: 애니메이션 및 인터랙션 추가
